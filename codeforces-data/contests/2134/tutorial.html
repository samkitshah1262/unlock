2134A - Painting With Two Colors

Hint 1

Check the parity of numbers aa, bb, and nn.

Solution

Let us first consider the bb blue cells. Since all bb cells appear in the final painting, for the coloring to be symmetric, they must be placed at the middle of the grid. Consequently, bb and nn must have the same parity.

Next, we examine two possible cases:

  • If b≥ab≥a, then the blue cells can fully cover all red cells. In this situation, the value of aa does not affect the final coloring.
  • If b<ab<a, then the red cells are not completely covered by the blue ones. Therefore, the red cells must also be placed in the middle of the grid, which means aa and nn must have the same parity as well.

Thus, by checking these conditions, we can determine whether a symmetric coloring exists. This requires only O(1)O(1) time.

Code (C++)
#include <iostream>
using namespace std;

int main() {
  ios::sync_with_stdio(false);
  cin.tie(0);
  int t;
  cin >> t;
  while (t--) {
    int n, a, b;
    cin >> n >> a >> b;
    if (a <= b) {
      cout << ((n % 2) == (b % 2) ? "YES\n" : "NO\n");
    } else {
      cout << ((n % 2) == (b % 2) && (n % 2) == (a % 2) ? "YES\n" : "NO\n");
    }
  }
}

Hint 1

Find an integer g>1g>1 and integers c1,c2,…,cnc1,c2,…,cn such that each ai+k⋅ciai+k⋅ci is divisible by gg.

Hint 2

Decide the value of gg only based on kk. There may be more than one possible choices.

Solution

The problem is equivalent to finding an array of nn non-negative integers c1,c2,…,cnc1,c2,…,cn such that 0≤ci≤k0≤ci≤k for 1≤i≤n1≤i≤n and

gcd(a1+k⋅c1,a2+k⋅c2,…,an+k⋅cn)>1gcd(a1+k⋅c1,a2+k⋅c2,…,an+k⋅cn)>1

The idea is as follows. If we can find an integer g>1g>1 and integers c1,c2,…,cnc1,c2,…,cn such that each ai+k⋅ciai+k⋅ci is divisible by gg (for 1≤i≤n1≤i≤n), then the GCD condition is satisfied.

There are (at least) two ways to construct such a solution. The simplest solution is to take g=k+1g=k+1 and define ci=aimod(k+1)ci=aimod(k+1). This works because

ai+k⋅ci≡ci+k⋅ci≡(k+1)⋅ci≡0(modk+1)ai+k⋅ci≡ci+k⋅ci≡(k+1)⋅ci≡0(modk+1)

Hence every term is divisible by k+1k+1.

Alternatively, you can also find the smallest integer xx such that gcd(x,k)=1gcd(x,k)=1, and set g=xg=x. Note that g≤29g≤29, since k≤109k≤109 and the product of all prime numbers from 22 to 2929 already exceeds 109109.

Then, for each aiai, we can add kk to it until it becomes divisible by gg. In fact, the value of cici can be determined explicitly (though it is not required) as

ai+k⋅ci≡0(modg)⟹ci≡(−ai⋅k−1)(modg)ai+k⋅ci≡0(modg)⟹ci≡(−ai⋅k−1)(modg)

where k−1k−1 is the modular inverse of kk modulo gg.

Thus, in either method, we can always construct a valid array so that the GCD of the resulting numbers is greater than 11.

2134C - Even Larger

Hint 1

Only subarrays with length equal to 22 or 33 are useful.

Hint 2

Propose a greedy algorithm to decide which element to perform an operation.

Solution

For convenience, if nn is even, we append an+1=0an+1=0 to the end of the array. Note that an+1an+1 cannot be modified.

The first key observation is the following:

Observation. If all subarrays [al,al+1,…,ar][al,al+1,…,ar] with the following properties satisfy the condition, then all subarrays satisfy the condition:

  • r=l+2r=l+2,
  • ll and rr are both odd.

Proof. Consider such subarrays where both ll and rr are odd:

∑l≤i≤ri evenai≥=≥=∑l≤i≤ri even(ai−1+ai+1)al+⎛⎝⎜⎜∑l+2≤i≤r−2i odd2ai⎞⎠⎟⎟+aral+⎛⎝⎜⎜∑l+2≤i≤r−2i oddai⎞⎠⎟⎟+ar∑l≤i≤ri oddai∑l≤i≤ri evenai≥∑l≤i≤ri even(ai−1+ai+1)=al+(∑l+2≤i≤r−2i odd2ai)+ar≥al+(∑l+2≤i≤r−2i oddai)+ar=∑l≤i≤ri oddai

The second inequality holds because all numbers are non-negative.

Thus, checking only these subarrays is sufficient: all other subarrays will automatically satisfy the condition because they involve fewer elements at odd positions. □◻

We will call these subarrays important in the following discussion.

Now we find the minimum number of operations. Instead of directly minimizing the number of operations, we can think in terms of constructing the final array bb after the operations. Our goal is to maximize the sum of the elements in bb.

Initially, we set bi=0bi=0 for all ii. Notice that it is never beneficial to reduce values at even indices. Therefore, we can safely set bi=aibi=ai for every even ii.

The task is then to determine the values of bibi at odd indices. It turns out that the following greedy algorithm works:

  • Repeatedly pick the smallest odd index ii such that bi<aibi<ai, and increasing bibi by 11 does not violate the condition for any important subarray. Then increment bibi by 11.

This greedy algorithm ensures that we maximize the sum of bb while preserving validity. It can be simulated in O(n)O(n) time.

Code (C++)
#include <algorithm>
#include <iostream>
#include <vector>
using namespace std;

int main() {
  ios::sync_with_stdio(false);
  cin.tie(0);
  int t;
  cin >> t;
  while (t--) {
    int n;
    cin >> n;
    vector<int> a(n);
    for (auto &i : a)
      cin >> i;

    vector<int> b(n + 1);
    long long ans = 0;
    for (int i = 0; i < n; i += 2) {
      int mn = a[i];
      if (i >= 2)
        mn = min(mn, a[i - 1] - b[i - 2]);
      if (i + 1 < n)
        mn = min(mn, a[i + 1]);
      b[i] = mn;
      ans += (a[i] - b[i]);
    }

    cout << ans << '\n';
  }
}

2134D - Sliding Tree

Hint 1

Consider the diameter of the tree. What do you discover?

Hint 2

Each sliding operation increases the diameter of the tree by at most 11.

Hint 3

Now about the construction. How to perform an operation at each step that increases the diameter by exactly 11?

Solution

We begin with the following observation:

Observation. Each sliding operation increases the diameter of the tree by at most 11.

Proof. Consider an operation (a,b,c)(a,b,c), any pair of indices (u,v)(u,v), and the unique path pp between uu and vv. If pp does not pass through bb, or if bb is an endpoint of pp, then the length of pp increases by at most 11 before and after the operation. Otherwise, let i1i1 and i2i2 denote the two neighbors of bb on the path pp. Then:

  • If i1=ai1=a and i2=ci2=c (or vice versa), the length of pp does not change.
  • If i1=ai1=a and i2≠ci2≠c (or vice versa), the length of pp increases by 11.
  • If i1≠ai1≠a and i2=ci2=c (or vice versa), the length of pp decreases by 11.
  • In all other cases, the length of pp does not change.

Thus, for any path in the tree, its length can increase by at most 11. Consequently, the diameter of the tree increases by at most 11 after a single operation. □◻

Let ll denote the diameter of the input tree. To transform it into a path graph (whose diameter is n−1n−1), it is clear that at least (n−1)−l(n−1)−l operations are required.

The remaining question is whether it is always possible to perform an operation at each step that increases the diameter by exactly 11. This turns out to be true. To see this, take any diameter of the tree and root the tree at one of its endpoints. In each operation, select an edge adjacent to the diameter. Suppose this edge is (u,v)(u,v), where uu lies on the diameter and vv does not. Then perform the operation with aa as the parent of uu (in the rooted tree), b=ub=u, and c=vc=v. One can verify that this increases the diameter by 11.

From the above construction, we conclude not only that the minimum number of operations is (n−1)−l(n−1)−l, but also that we can explicitly construct a sequence of operations that transforms the given tree into a path graph with this minimum number of moves.

Finally, finding the first sliding operation can be done in O(n)O(n) time.

Code (C++)
#include <algorithm>
#include <functional>
#include <iostream>
#include <vector>
using namespace std;

int main() {
  ios::sync_with_stdio(false);
  cin.tie(0);
  int t;
  cin >> t;
  while (t--) {
    int n;
    cin >> n;
    vector<vector<int>> G(n);
    for (int i = 0; i < n - 1; i++) {
      int u, v;
      cin >> u >> v;
      u--, v--;
      G[u].push_back(v);
      G[v].push_back(u);
    }

    vector<int> dist(n), p(n);
    function<void(int, int)> dfs = [&](int now, int par) {
      p[now] = par;
      for (int i : G[now]) {
        if (i != par) {
          dist[i] = dist[now] + 1;
          dfs(i, now);
        }
      }
    };

    dist[0] = 0;
    dfs(0, -1);
    int x = max_element(dist.begin(), dist.end()) - dist.begin();
    dist[x] = 0;
    dfs(x, -1);
    int y = max_element(dist.begin(), dist.end()) - dist.begin();

    if (dist[y] == n - 1) {
      cout << -1 << '\n';
    } else {
      vector<int> on_diameter(n);
      int now = y;
      while (now != -1) {
        on_diameter[now] = 1;
        now = p[now];
      }
      int a = -1, b = -1, c = -1;
      for (int u = 0; u < n; u++) {
        for (int v : G[u]) {
          if (on_diameter[u] && !on_diameter[v]) {
            a = p[u], b = u, c = v;
            break;
          }
        }
        if (a != -1)
          break;
      }
      cout << a + 1 << ' ' << b + 1 << ' ' << c + 1 << '\n';
    }
  }
}

2134E - Power Boxes

Hint 1

An intuitive starting point may be to perform a throw query on every coordinate ii. For which ii we can decide the value of aiai immediately?

Hint 2

For those incides whose aiai is unknown, try to perform some swap queries on them to get the values of aiai for them.

Hint 3

No two unknown indices are adjacent.

Solution

An intuitive starting point is to perform a throw query on every coordinate ii (1≤i≤n1≤i≤n). Let didi denote the total number of jumps the ball makes when starting from coordinate ii. For convenience, define dn+1=dn+2=0dn+1=dn+2=0.

Notice that for 1≤i≤n1≤i≤n, if di+1≠di+2di+1≠di+2, then we can immediately determine aiai: if di=di+1+1di=di+1+1, then ai=1ai=1; otherwise ai=2ai=2.

The difficult part of the problem is when di+1=di+2di+1=di+2. We call an index ii \textbf{unknown} if di+1=di+2di+1=di+2. In such cases, we can deduce didi without issuing a query, thus saving it for later use. The following observation is crucial:

Observation. No two unknown indices are adjacent.

Proof. If ii is unknown, then di=di+1+1di=di+1+1, hence di≠di+1di≠di+1. Therefore, i−1i−1 cannot be unknown. □◻

To exploit this fact, we adopt the following strategy: After computing all didi, for each unknown index ii (in increasing order), we swap the boxes at positions ii and i+1i+1, and then perform a throw query at coordinate i+1i+1. Since index i+1i+1 is not unknown, this reveals aiai. The special case is i=ni=n: here, we swap boxes n−1n−1 and nn, then perform a throw query on coordinate n−1n−1 to obtain anan.

To sum up:

  • For ii from nn down to 11: perform a throw query on coordinate ii. If di+1=di+2di+1=di+2, skip the query for ii and mark ii as unknown (but still compute didi). Otherwise, determine both aiai and didi.
  • For each ii from 11 to n−1n−1: if ii is unknown, swap box ii with box i+1i+1 and perform a throw query at coordinate i+1i+1. This reveals aiai.
  • Finally, swap n−1n−1 and nn, and perform a throw query on coordinate n−1n−1 to determine anan.

For every unknown index we use 22 queries, and for each known index we use 11. Since no two unknown indices are adjacent, there are at most ⌈n2⌉⌈n2⌉ unknown indices. Hence, the total number of queries is at most ⌈3n2⌉⌈3n2⌉.

#include <iostream>
#include <vector>
using namespace std;

int main() {
  ios::sync_with_stdio(false);
  cin.tie(0);
  int t;

    int n;
    cin >> n;

    vector<int> a(n);
    auto swap = [&](int x) {
      cout << "swap " << x + 1 << endl;
    };
    auto throw_ball = [&](int x) {
      cout << "throw " << x + 1 << endl;
      int ret;
      cin >> ret;
      return ret;
    };
    auto answer = [&]() {
      cout << "!";
      for (auto i : a)
        cout << ' ' << i;
      cout << endl;
    };

    vector<int> jumps(n + 2);
    for (int i = n - 1; i >= 0; i--) {
      if (jumps[i + 1] == jumps[i + 2]) {
        jumps[i] = jumps[i + 1] + 1;
      } else {
        jumps[i] = throw_ball(i);
        if (jumps[i] == jumps[i + 1] + 1)
          a[i] = 1;
        else
          a[i] = 2;
      }
    }
    for (int i = 0; i + 1 < n; i++) {
      if (a[i] == 0) {
        swap(i);
        int jumps_i = throw_ball(i + 1);
        if (jumps_i == jumps[i + 2] + 1)
          a[i] = 1;
        else
          a[i] = 2;
      }
    }
    swap(n - 2);
    int jumps_last = throw_ball(n - 2);
    if (jumps_last == 2)
      a[n - 1] = 1;
    else
      a[n - 1] = 2;
    answer();
  }
}

2134F - Permutation Oddness

Thanks to Misuki for coming up with a better solution! My original solution has a much larger constant.

Hint 1

Consider two groups of numbers; one is 0,20,2 and another is 1,31,3. What do you discover?

Hint 2

Consider the segments formed by 0,20,2 (same for 1,31,3). For each ii and jj, find the number of ways to arrange them into ii segments so that the total number of adjacent pairs inside all segments contribute oddness jj.

Hint 3

Enumerate all binary strings consisting of c0c0 zeros and c2c2 twos. Then split these strings into several substrings.

Hint 4

Brute force over the number of runs of the binary string. Find the contribution to the count.

Solution

Consider any rearrangement bb. We color all elements bibi with value 00 or 22 as red, and those with value 11 or 33 as blue. Then the array can be viewed as alternating red and blue segments.

We observe the following:

  • Adjacent pairs within a red segment contribute 00 or 22 to the oddness. The same holds for blue segments.
  • Adjacent pairs between red and blue segments contribute exactly 11 to the oddness.
  • The number of red and blue segments differs by at most 11.

Hence, if we can calculate the following:

  • ri,jri,j: number of ways to arrange c0c0 zeros and c2c2 twos into ii segments, so that the total number of adjacent pairs inside all segments contribute oddness jj.
  • Similarly define bi,jbi,j for the blue digits (11 and 33)

Then the final answer can be computed in O(n3)O(n3) by performing convolutions between the arrays pairs (ri,bi),(ri+1,bi),(ri,bi+1)(ri,bi),(ri+1,bi),(ri,bi+1).

Now about computing ri,jri,j. This reduces to counting tuples of ii binary strings formed from c0c0 zeros and c2c2 twos, such that the total number of alternating adjacent pairs inside them equals jj.

The idea is as follows. First, enumerate all binary strings consisting of c0c0 zeros and c2c2 twos. Then split these strings into several substrings.

Suppose a binary string has ll runs. Let l1=⌈l2⌉l1=⌈l2⌉ be the number of runs of 00, and l2=l−l1l2=l−l1 the number of runs of 22 (or vice versa). The number of such binary strings is

(c0−1l1−1)⋅(c2−1l2−1)(c0−1l1−1)⋅(c2−1l2−1)

In such strings, there are l−1l−1 adjacent pairs with different numbers, and c0+c2−lc0+c2−l adjacent pairs with the same numbers.

Now split the string into several substrings. Assume we choose k1k1 positions among the l−1l−1 differing adjacencies, and k2k2 positions among the c0+c2−lc0+c2−l equal adjacencies. The number of ways to do this is

(l−1k1)⋅(c0+c2−lk2)(l−1k1)⋅(c0+c2−lk2)

This partition produces k1+k2+1k1+k2+1 substrings, each contributing oddness 2⋅(l−1−k1)2⋅(l−1−k1).
Thus, the contribution to rk1+k2+1,2⋅(l−1−k1)rk1+k2+1,2⋅(l−1−k1) is

(c0−1l1−1)⋅(c2−1l2−1)⋅(l−1k1)⋅(c0+c2−lk2)(c0−1l1−1)⋅(c2−1l2−1)⋅(l−1k1)⋅(c0+c2−lk2)

Enumerating all possible tuples (l,k1,k2)(l,k1,k2) suffices to compute all ri,jri,j. This computation takes O(n3)O(n3) time.

By the same process, we can compute bi,jbi,j in O(n3)O(n3). Finally, convolving the results yields the number of distinct permutations for each oddness value. The entire algorithm runs in O(n3)O(n3) time.

  M(T x = 0) {

    if (v < 0)
      v += MOD;
  }
  explicit operator T() const { return v; }
  bool operator==(const M &b) const { return v == b.v; }
  bool operator!=(const M &b) const { return v != b.v; }

  M operator+(M b) { return M(v + b.v); }
  M operator-(M b) { return M(v - b.v); }
  M operator*(M b) { return M((long long)v * b.v % MOD); }

        ans *= a;

    for (auto &i : cnt)
      cin >> i;
    int n = cnt[0] + cnt[1] + cnt[2] + cnt[3];

    vector<vector<Mod>> r(n + 1, vector(n + 1, Mod(0))),
        b(n + 1, vector(n + 1, Mod(0)));

    auto fill_table = [&](int x, int y, vector<vector<Mod>> &table) {
      for (int i = 2; i <= x + y; i++) {
        vector<pair<int, int>> segments;
        if (i & 1)
          segments = {{i / 2, i - i / 2}, {i - i / 2, i / 2}};
        else
          segments = {{i / 2, i / 2}, {i / 2, i / 2}};
        for (auto [l1, l2] : segments) {
          if (x < l1 || y < l2)
            continue;
          auto mul = binom(x - 1, l1 - 1) * binom(y - 1, l2 - 1);
          int different = i - 1, same = (x + y) - 1 - different;
          for (int j = 0; j <= different; j++)
            for (int k = 0; k <= same; k++) {
              auto mul2 = binom(different, j) * binom(same, k);
              int c = different - j;
              table[j + k + 1][c] += mul * mul2;
            }
        }
      }
    };

    fill_table(cnt[0], cnt[2], r);
    fill_table(cnt[1], cnt[3], b);

    vector<Mod> ans(2 * (n - 1) + 1);
    for (int i = 1; i <= (n + 1) / 2; i++)
      for (int j = 0; j <= n; j++)
        for (int k = 0; k <= n; k++) {
          int oddness = (j + k) * 2 + (i * 2 - 1);
          if (oddness >= 0 && oddness <= 2 * (n - 1))
            ans[oddness] += r[i][j] * b[i][k] * 2;
        }
    for (int i = 1; i <= (n + 1) / 2 && i + 1 <= n; i++)
      for (int j = 0; j <= n; j++)
        for (int k = 0; k <= n; k++) {
          int oddness = (j + k) * 2 + i * 2;
          if (oddness >= 0 && oddness <= 2 * (n - 1)) {
            ans[oddness] += r[i][j] * b[i + 1][k];
            ans[oddness] += r[i + 1][j] * b[i][k];
          }
        }

    for (int i = 0; i <= 2 * (n - 1); i++)
      cout << ans[i].v << " \n"[i == 2 * (n - 1)];
  }
