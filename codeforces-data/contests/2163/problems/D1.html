This is the Easy version of the problem. The difference between the versions is that in this version, you can make at most max(300,⌈n2⌉+2)max(300,⌈n2⌉+2) queries. You can hack only if you solved all versions of this problem.

This problem is interactive.

There is a permutation∗∗ pp of the integers from 00 to n−1n−1 hidden from you. Additionally, you are given qq ranges [l1,r1],[l2,r2],…,[lq,rq][l1,r1],[l2,r2],…,[lq,rq] where 1≤li≤ri≤n1≤li≤ri≤n.

You need to discover the maximum MEXMEX among the values of pp in the qq ranges that are given to you in the input. Formally, you must discover the value of maxqi=1MEX([pli,pli+1,…,pri])maxi=1qMEX⁡([pli,pli+1,…,pri])††. To do this, you may make at most max(300,⌈n2⌉+2)max(300,⌈n2⌉+2) queries of the following form:

  • Choose any two integers 1≤l≤r≤n1≤l≤r≤n and you will receive the value MEX([pl,pl+1,…,pr])MEX⁡([pl,pl+1,…,pr]).

∗∗A permutation of the integers from 00 to n−1n−1 is a sequence of nn elements where every integer from 00 to n−1n−1 appears exactly once. For example, the sequence [0,3,1,2][0,3,1,2] is a permutation, but the sequence [0,0,2,1][0,0,2,1] is not.

††The MEXMEX of a sequence is defined as the smallest non-negative integer that does not appear in that sequence. For example, MEX([0,0,1,3])=2MEX⁡([0,0,1,3])=2 and MEX([1,2,2])=0.MEX⁡([1,2,2])=0.

Input

Each test contains multiple test cases. The first line contains the number of test cases tt (1≤t≤1001≤t≤100). The description of the test cases follows.

The first line of each test case contains exactly two integers nn and qq (4≤n≤1044≤n≤104, 1≤q≤3⋅1051≤q≤3⋅105) — the size of the permutation and the number of ranges, respectively.

The ii-th of the next qq lines contains two integers li,rili,ri (1≤li≤ri≤n1≤li≤ri≤n).

It is guaranteed that the sum of nn and qq do not exceed 104104 and 3⋅1053⋅105 respectively over all test cases.

Additional Constraint: it is guaranteed that no range is repeated in the same test case.

Interaction

To ask a query, output a line in the following format (without the quotes):

  • "? ll rr" (1≤l≤r≤n1≤l≤r≤n)

The jury will return a single integer, the value of MEX([pl,pl+1,…,pr])MEX⁡([pl,pl+1,…,pr]).

When you have found the answer, output a single line in the following format:

  • "! xx" (0≤x≤n0≤x≤n).

After that, proceed to process the next test case or terminate the program if it was the last test case. Printing the answer does not count as a query.

The interactor is not adaptive, meaning that the values of the permutation are known before the participant asks the queries.

If your program makes more than max(300,⌈n2⌉+2)max(300,⌈n2⌉+2) queries, your program should immediately terminate to receive the verdict Wrong Answer. Otherwise, you can get an arbitrary verdict because your solution will continue to read from a closed stream.

After printing a query do not forget to output the end of line and flush the output. Otherwise, you may get the Idleness Limit Exceeded verdict. To do this, use:

  • fflush(stdout) or cout.flush() in C++
  • System.out.flush() in Java;
  • flush(output) in Pascal;
  • stdout.flush() in Python;
  • see the documentation for other languages.

Hacks

For hacks, use the following format.

The first line of the input should contain a single integer tt (1≤t≤1001≤t≤100) — the number of test cases.

The first line of each test case should contain exactly two integers nn and qq (4≤n≤1044≤n≤104, 1≤q≤3⋅1051≤q≤3⋅105) — the size of the permutation and the number of ranges you are given, respectively.

The second line should contain nn integers p1,p2,…,pnp1,p2,…,pn — where pipi is the ii-th element of the permutation. It should hold that pp is a permutation containing integers 00 to n−1n−1.

The ii-th of the next qq lines should contain two integers lili, riri (1≤li≤ri≤n1≤li≤ri≤n).

It should hold that the sum of nn and qq do not exceed 104104 and 3⋅1053⋅105 respectively over all test cases.

Additional Constraint: no range should be repeated in the same test case.
