This is a run-twice (communication) problem.

There are two players: Player A and Player B. The jury will first interact with player A. After player A ends their interaction, the jury will interact with player B. Note that player A and player B may not directly pass information to each other; both players are only able to send information or receive information from the jury, but they may agree on the strategy they will use to communicate.

The jury has a binary grid GG of nn rows and nn columns (each cell of this grid has value either 00 or 11). Row 11 is the top-most row, and column 11 is the left-most column. The connectivity of this grid is defined as 11 if there is a path going left, right, up, or down going through only cells with value 11 connecting each pair of cells (i1,j1)(i1,j1) and (i2,j2)(i2,j2) with Gi1,j1=Gi2,j2=1Gi1,j1=Gi2,j2=1. Note that moving diagonally is not allowed. It is guaranteed that there exists at least one cell with value of 11 in this grid.

The jury first interacts with player A. The jury will give player A the grid GG. After inspecting the grid, player A must determine two integers rr and cc and send them to the jury. At the start of player B's interaction, player B will receive the values of all cells in the rr'th row and all cells in the cc'th column from the jury. Note that player B is not given the values of rr and cc.

Player A wants to ensure player B can determine the connectivity of GG. Your task is to act as both players and find a strategy so that player B is able to determine the connectivity correctly. Note that the communicator of this task is not adaptive – that is, the grid given to you on the first run will be the same as the grid used to evaluate the connectivity.

Input

Your code will be ran exactly two times on each test. On the first run, you will be Player A, and on the second Player B.

First Run Input

The first line of the input contains the string first. The purpose of this is so your program recognizes that this is its first run, and it should act as Player A.

Each test contains multiple test cases. The first line contains the number of test cases tt (1≤t≤1041≤t≤104). The description of the test cases follows.

The first line of each test case contains two integers nn and CC (2≤n≤1000,0≤C≤12≤n≤1000,0≤C≤1) – the size of the grid and the connectivity respectively.

The following nn lines contain information about the grid. The iith of these lines contains a binary string GiGi of length nn, indicating the ii-th row of the grid.

It is guaranteed that:

  • The sum of n2n2 does not exceed 2⋅1062⋅106 over all test cases
  • The value of CC matches with the information in the grid – that is, if C=1C=1, then the grid has connectivity 11, and if C=0C=0, then the grid has connectivity 00.
  • Each grid has at least one 11 in the input.

Second Run Input

The first line of the input contains the string second. The purpose of this is so your program recognizes that this is its second run, and it should act as Player B.

Each test contains multiple test cases. The first line contains the number of test cases tt (1≤t≤1041≤t≤104). The description of the test cases follows. Note that this number is equal to tt from the first run input.

The first line of each test case contains exactly one integer nn — the size of the grid given on the ii'th test case of the first run input.

The second line of each test case contains a binary string Gr,1Gr,2…Gr,nGr,1Gr,2…Gr,n — the contents in row rr of the grid. Note that the integer rr is sent by player A to the jury at the end of their interaction of the ii'th test case

The third line of each test case contains a binary string G1,cG2,c…Gn,cG1,cG2,c…Gn,c — the contents in column cc of the grid. Note that the integer cc is sent by player A to the jury at the end of their interaction of the ii'th test case.

Hacks

To make hacks, use the following format:

The first line should contain exactly one integer tt (1≤t≤104)(1≤t≤104) — the number of grids. Then, tt blocks of input should follow.

The first line of each block of input must contain a single integer nn (2≤n≤1000)(2≤n≤1000) — the size of the grid the jury will choose.

Each of the next nn lines should contain a binary string of size nn. The ii'th of these lines should contain Gi,1Gi,2…Gi,nGi,1Gi,2…Gi,n — the contents of the ii'th row of the grid the jury will choose.

There must exist at least a single cell with value of 11 in each grid, and the sum of n2n2 over all test cases should not exceed 2⋅1062⋅106.

Note that the connectivity of each grid is determined by the jury, and you do not need to output it for a hack.

Output

For the first run, for each test case, output two integers rr and cc (1≤r,c≤n1≤r,c≤n). This indicates that you want the second run to receive the rr-th row and the cc-th column of the grid.

For the second run, for each test case, output an integer CC (0≤C≤10≤C≤1) – the connectivity of the grid.

For the sake of this example, suppose Player A and Player B have agreed to some strategy where if the connectivity is 11, then player A should send a row and a column that both end with 00. In this case, row 22 and column 22 satisfies this strategy. Note that this is an example strategy for the sake of demonstration, and using this strategy will not work for all cases.

Then, take a look at the second run. Now, we are Player B. We receive that n=2n=2 and that the chosen row has values r=[1,0]r=[1,0] and the chosen column has values c=[1,0]c=[1,0]. Since the last number of each row is 00, player B uses the agreed upon strategy to determine that the connectivity is 11.
