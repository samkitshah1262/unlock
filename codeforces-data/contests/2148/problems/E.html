Farmer John has an array aa containing nn positive integers and an integer kk.

Let a[l,r]a[l,r] be a subarray∗∗ of aa. He performs the following procedure to independently determine if subarray a[l,r]a[l,r] is awesome:

  • Initially, FJ has kk empty multisets, numbered from 11 to kk.
  • Then, for each element aiai (1≤i≤n1≤i≤n) in aa:
    • If l≤i≤rl≤i≤r (that is, aiai is in the subarray a[l,r]a[l,r]), he places aiai in multiset 11,
    • Otherwise, he places aiai into any multiset he wants (which may be multiset 11).
  • Subarray a[l,r]a[l,r] is awesome if there is some way for him to place elements such that, for every value vv, all multisets contain the same number of elements with value vv. In other words, he wants to make all multisets contain the exact same elements (ignoring ordering).

Output the number of awesome subarrays.

∗∗For array aa of size nn and integers 1≤l≤r≤n1≤l≤r≤n, the subarray a[l,r]a[l,r] denotes the array consisting of the elements al,…,aral,…,ar, in order.

Input

The first line contains an integer tt (1≤t≤10001≤t≤1000) — the number of test cases.

The first line of each test case contains two integers nn and kk (2≤k≤n≤2⋅1052≤k≤n≤2⋅105).

The following line contains nn space-separated integers a1,a2,…,ana1,a2,…,an (1≤ai≤n1≤ai≤n).

It is guaranteed that the sum of nn over all test cases does not exceed 2⋅1052⋅105.

Output

For each testcase, output one integer on a new line: the number of awesome subarrays.

Note

Test case 1: n=3n=3, a=[1,1,1]a=[1,1,1].

For k=2k=2, you cannot finish with the same number of 11's in both multisets, so no subarray works.

Test case 2: n=4n=4, a=[1,2,1,2]a=[1,2,1,2].

For k=2k=2, the final state must give each multiset exactly one 11 and one 22. Thus a valid subarray can contain at most one 11 and at most one 22.
