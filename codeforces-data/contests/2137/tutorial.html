2137A - Collatz Conjecture

Problem Credits: Proof_by_QED

Solution

The reverse of the Collatz conjecture can be written as follows:

  • Multiply xx by 22 (if 2x2x was even)
  • Subtract 11 from xx, then divide by 33 (if x−13x−13 is an integer and is odd).

In this problem, we can ignore the second type completely, because 2x2x is always an integer and is always even. We can simply print out x⋅2kx⋅2k as our answer.

Code
import sys
input = sys.stdin.readline
for _ in range(int(input())):
    n,k = map(int,input().split())
    print(k<<n)

2137C - Maximum Even Sum

Problem Credits: Proof_by_QED

Solution

First, observe that in order to maximize the sum of two positive numbers given that their product must stay constant, it is optimal to minimize one number as much as possible (and in the process, maximize the other).

Let's do casework on the parity of aa and bb:

  • aa, bb are both even: in this case, we cannot choose a kk such that bkbk is odd, as aa will stay even, and a+ba+b will be odd. The optimal kk here is b2b2.
  • aa is even, but bb is odd: In this case, we should print −1−1, as it is impossible to make a+ba+b even.
  • aa is odd, bb is divisible by 22 and not 44: This case is also impossible. No matter what kk we choose, one of aa and bb will always be odd.
  • aa is odd, bb is divisible by 44: Even though a+ba+b is initially odd, it is possible to make a+ba+b even in this case by choosing a kk so that kk and bkbk are both even. Once again, the optimal kk to use is b2b2.
  • aa and bb are both odd: In this case, the optimal kk to use is bb. Note that a+ba+b will always be even no matter what kk is chosen.
Code
import sys
input = sys.stdin.readline
for _ in range(int(input())):
    b,a = map(int,input().split())
    sol = -1
    if (a + b) & 1 == 0: sol = a + b
    if a % 2 == 1 and b % 2 == 1: sol = max(sol, a * b + 1)
    elif a % 2 == 0 and (a % 4 == 0 or b % 2 == 0): sol = max(sol, 2 + (a * b) // 2)
    print(sol)

2137D - Replace with Occurrences

Problem Credits: Proof_by_QED

Solution

First, we should observe that the number of occurrences of xx in bb must be a multiple of xx. This is true because if we have bi=kbi=k, then there are kk occurrences of aiai in the original array. Suppose we group up all equal elements in the array aa. If the number of occurrences of bibi is not a multiple of bibi, then there is at least one group which doesn't have bibi elements. This is a contradiction.

If the number of occurrences of bibi is a multiple of bibi for all ii, then we can construct as follows:

  • keep a varible z=1z=1.
  • For each xx from 11 to nn, iterate through the all indices ii with bi=xbi=x. Label the first xx numbers as zz, then increase zz by 11. Continue until all indices are labeled.

This solution works in O(n)O(n).

2137E - Mexification

Problem Credits: Proof_by_QED

Solution

Let's first determine how to run the operation a single time.

Let mm be the MEX of the array, and let occiocci be the number of occurrences of the number ii. We have these cases:

  • If ai>mai>m, then aiai will become mm.
  • If occai=1occai=1 and ai<mai<m, then aiai will still be itself after the operation. This is because after we remove aiai from the array, aiai will be the new MEX.
  • if occai>1occai>1 and ai<mai<m, then aiai will become mm. This is because even after removing aiai, there will still be at least one more occurrence of aiai, so aiai will simply become the MEX of the whole array.

From this, we notice that numbers with frequency 11 and less than the MEX will not change, while other numbers will map to the MEX. We now discuss several possible cases:

  • If aa is a permutation of [0,1,…,n−1][0,1,…,n−1], then doing operations on aa will not change anything about the array.
  • Otherwise, let xx be the smallest number with occx≠1occx≠1. If occx=0occx=0, then xx is the MEX of aa. After the first operation, all numbers greater than xx will get mapped to xx while all numbers less than xx will map to themselves (because occi=1occi=1 for all i≤ni≤n. Now, all occurrences of xx (if there are more than 11) will map to x+1x+1. The array will then continuously alter between these two states. Be careful of the edge case [0,1,…,n−2,n][0,1,…,n−2,n], where it will map to the permutation case and not change again.
  • Otherwise, occx>1occx>1. Let mm be the MEX of aa. After the first operation, all numbers less than xx will map to itself, while all occurrences of xx will map to mm. This array is now the case of the previous one (where the smallest xx with occx≠1occx≠1 will have occx=0occx=0), meaning that it will continuously alter between two states.

In all cases, after at most two operations, the array will only alter between two states after at most one operation. So it is possible to simulate the operation min(k,kMOD2+2)min(k,kMOD2+2) times and print out the sum.

2137F - Prefix Maximum Invariance

Problem Credits: Proof_by_QED

Solution

What is the condition for being able to make xi=yixi=yi? It is always possible if xi=yixi=yi already. If xi>yixi>yi, it's only possible if aiai is not already a prefix maximum. If xi<yixi<yi, it's only possible if there is a prefix maximum in the array before ii that is at least equal to yiyi.

Let's formalize the conditions for the last two cases. If xi>yixi>yi, then there must exist an index j<ij<i such that xj>xixj>xi (as this guarantees aiai to not be a prefix maximum). If xi<yixi<yi, then there must exist an index j<ij<i such that xj≥yixj≥yi, as this ensures you can change xixi to yiyi. Let's calculate the correct jj for each ii in the original array aa. This can be done in a variety of data structures. I used a set, but it is possible to use a segment tree or a stack as well.

Finally, to find the sum over all subarrays in aa, for each index ii we need to calculate the number of subarrays containing ii where we can make ai=biai=bi. If xi=yixi=yi, all subarrays containing ii counts, so we add (i)(n−i+1)(i)(n−i+1) to our answer. Otherwise, let previprevi be the number we calculated before. We can add (i−previ+1)(n−i+1)(i−previ+1)(n−i+1) to our answer.

Code
import sys
input = sys.stdin.readline
class SparseTarble():
    def __init__(self,arr,op=min):
        self.n = len(arr)
        self.log = self.n.bit_length()
        self.PRE = [0 for _ in range(self.log * self.n)]
        for i in range(self.n):
            lsb = (i+1) & (-i-1)
            v = lsb.bit_length() - 1
            jo = self.n * v
            self.PRE[jo + i] = arr[i]
            for j in range(i+1,min(self.n,i+lsb)):
                self.PRE[jo + j] = op(self.PRE[jo+j-1],arr[j])
            for j in range(i-1, i - lsb, -1):
                self.PRE[jo + j] = op(arr[j],self.PRE[jo+j+1])
        self.op = op
    def query(self,l,r):
        assert(l < r)
        bit = (r ^ l).bit_length() - 1
        return self.op(self.PRE[bit * self.n + l], self.PRE[bit * self.n + r - 1])
for _ in range(int(input())):
    n = int(input())
    a = list(map(int,input().split()))
    b = list(map(int,input().split()))
    sq = SparseTarble(a,max)
    sol = 0
    for i in range(n):
        if a[i] == b[i]: sol += (n * n + n) // 2; continue
        lo = -1; hi = i
        while lo + 1 < hi:
            mid = (lo + hi) // 2
            if sq.query(mid,i) >= max(a[i],b[i]): lo = mid
            else: hi = mid
        lo+=1
        sol += lo * (n - i)
    print(sol)

2137G - Cry Me a River

Problem Credits: SpyrosAliv

Solution

Let dp1(i)dp1(i) be a boolean on whether Cry will win if the token is on node ii and it is currently Cry's turn, and let dp2(i)dp2(i) be a boolean on whether Cry will win a game if we are on node ii and it is currently River's turn. Initially, dp1(i)=1dp1(i)=1 and dp2(i)=1dp2(i)=1 for all 1≤i≤n1≤i≤n. Assuming we maintain the correct dpdp values all the time, queries of the second type is easily answered by looking at the dp state.

How can we maintain the correct dpdp values after a node is painted red? Consider BFS on the reverse graph. Suppose node uu was turned red. First, node uu becomes winning for River immediately no matter whose turn it is. Now, if a node becomes losing on Cry's turn, all nodes that are adjacent to that node in the reverse graph becomes winning on River's turn (as River can move the node to a node that loses on Cry's turn). If a node becomes losing on River's turn, consider the set of nodes adjacent on the reverse graph. If all nodes adjacent to that node (on the original graph) is losing on River's turn, then that node is also losing on Cry's turn, as no matter which node he decides to move to, it will be going to a lost node.

Keeping track of the DP states seems like it will be O(n2)O(n2) complexity. However, since each node is visited only twice (once for updating dp1dp1 and once for updating dp2dp2, this solution will amortize to O(n+m)O(n+m). Keep note that we should maintain a third array that counts how many adjacent nodes are losing on River's turn, so we don't have to check manually every time.

Code
import sys
input = sys.stdin.readline
for _ in range(int(input())):
    n,m,q = map(int,input().split())
    G = [[] for _ in range(n+1)]
    A = [1 for _ in range(n+1)]
    B = [0 for _ in range(n+1)]
    C = [0 for _ in range(n+1)]
    for _ in range(m):
        a,b = map(int,input().split())
        G[b].append(a)
        C[a] += 1
    for _ in range(q):
        a,b = map(int,input().split())
        if a == 2:
            if A[b]: print("yes")
            else: print("no")
        else:
            q = []
            if A[b] == 1: q.append((b,0))
            if B[b] == 0: q.append((b,1)); B[b] = 1
            for x,t in q:
                if t:
                    for c in G[x]:
                        C[c] -= 1
                        if C[c] == 0:
                           q.append((c,0))
                else:
                    if A[x] == 0: continue
                    A[x] = 0
                    for c in G[x]:
                        if B[c]: continue
                        B[c] = 1
                        q.append((c,1))
