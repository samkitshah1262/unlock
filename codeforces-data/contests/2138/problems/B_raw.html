B. Antiamuny Wants to Learn Swap
time limit per test
2 seconds
memory limit per test
512 megabytes
input
standard input
output
standard output

For an array bb of length mm, you may perform the following two operations:

  1. Select an index 1≤i≤m−11≤i≤m−1. Then, swap the values of bibi and bi+1bi+1.
  2. Select an index 1≤i≤m−21≤i≤m−2. Then, swap the values of bibi and bi+2bi+2.
However, you can only perform operation 22 at most once.

We define f(b)f(b) as the minimum number of operations (using both operation 1 and operation 2) required to sort array bb in non-decreasing order, and g(b)g(b) as the minimum number of operations required to sort array bb in non-decreasing order using only operation 1.

The array bb is perfect if f(b)=g(b)f(b)=g(b). In other words, the ability to use operation 2 does not reduce the number of operations required to sort array bb compared to using only adjacent swaps.

You are given a permutation aa of length nn∗∗, and must answer qq queries. Each query consists of two integers ll and rr (1≤l≤r≤n1≤l≤r≤n), representing the subarray a[l…r]a[l…r]††. For each query, determine whether the subarray a[l…r]a[l…r] is perfect.

∗∗A permutation of length nn is an array consisting of nn distinct integers from 11 to nn in arbitrary order. For example, [2,3,1,5,4][2,3,1,5,4] is a permutation, but [1,2,2][1,2,2] is not a permutation (22 appears twice in the array), and [1,3,4][1,3,4] is also not a permutation (n=3n=3 but there is 44 in the array).

††The subarray a[l…r]a[l…r] includes all elements from index ll to rr, i.e., [al,al+1,al+2,…,ar][al,al+1,al+2,…,ar].

Input

Each test contains multiple test cases. The first line contains the number of test cases tt (1≤t≤5⋅1041≤t≤5⋅104). The description of the test cases follows.

The first line of each test case contains two integers nn, qq (1≤n,q≤5⋅1051≤n,q≤5⋅105) — the length of array aa and the number of queries.

The second line of each test case contains nn integers a1,a2,…,ana1,a2,…,an (1≤ai≤n1≤ai≤n) — the elements in permutation aa.

Each of the next qq lines contains two integers ll and rr (1≤l≤r≤n1≤l≤r≤n) — the left and right endpoints of the queried subarray.

It is guaranteed that both the sum of nn and the sum of qq over all test cases do not exceed 5⋅1055⋅105.

Output

For each test case, output "YES" if queried subarray a[l…r]a[l…r] is perfect, and "NO" otherwise.

You can output the answer in any case (upper or lower). For example, the strings "yEs", "yes", "Yes", and "YES" will be recognized as positive responses.

Note

In the first test case:

  • Query 1: a[1…2]=[1,5]a[1…2]=[1,5] is already sorted in increasing order. Thus, f(a[1…2])=g(a[1…2])=0f(a[1…2])=g(a[1…2])=0 and the subarray a[1…2]a[1…2] is perfect.
  • Query 2: a[1…5]=[1,5,4,3,2]a[1…5]=[1,5,4,3,2]. f(a[1…5])=4f(a[1…5])=4 as we can sort the array using the following sequence of operations:

    [1,5,4,3,2]−→−op2[1,3,4,5,2]−→−op1[1,3,4,2,5]−→−op1[1,3,2,4,5]−→−op1[1,2,3,4,5][1,5,4,3,2]→op2[1,3,4,5,2]→op1[1,3,4,2,5]→op1[1,3,2,4,5]→op1[1,2,3,4,5]

    On the other hand, g(a[1…5])=6g(a[1…5])=6 as it requires at least 66 adjacent swaps to sort a[1…5]a[1…5]. Since f(a[1…5])≠g(a[1…5])f(a[1…5])≠g(a[1…5]), the subarray a[1…5]a[1…5] is not perfect.

  • Query 3: a[3…5]=[4,3,2]a[3…5]=[4,3,2]. f(a[3…5])=1f(a[3…5])=1 as we can sort the array using the following sequence of operations:

    [4,3,2]−→−op2[2,3,4][4,3,2]→op2[2,3,4]

    On the other hand, g(a[3…5])=3g(a[3…5])=3 as it requires at least 33 adjacent swaps to sort a[3…5]a[3…5]. Since f(a[3…5])≠g(a[3…5])f(a[3…5])≠g(a[3…5]), the subarray a[3…5]a[3…5] is not perfect.
