F. Counting Necessary Nodes
time limit per test
2 seconds
memory limit per test
512 megabytes
input
standard input
output
standard output

A quadtree is a tree data structure in which each node has at most four children and accounts for a square-shaped region.

Formally, for all tuples of nonnegative integers k,a,b≥0k,a,b≥0, there exists exactly one node accounting for the following region∗∗.

[a⋅2k,(a+1)⋅2k]×[b⋅2k,(b+1)⋅2k][a⋅2k,(a+1)⋅2k]×[b⋅2k,(b+1)⋅2k]

All nodes whose region is larger than 1×11×1 contain four children corresponding to the regions divided equally into four, and the nodes whose region is 1×11×1 correspond to the leaf nodes of the tree.

A small subset of the regions accounted for by the nodes is shown. The relatively darker regions are closer to leaf nodes.

The Frontman hates the widespread misconception, such that the quadtree can perform range queries in O(logn)O(log⁡n) time when there are nn leaf nodes inside the region. In fact, sometimes it is necessary to query much more than O(logn)O(log⁡n) regions for this, and the time complexity is O(n)O(n) in some extreme cases. Thus, the Frontman came up with this task to educate you about this worst case of the data structure.

The pink soldiers have given you a finite region [l1,r1]×[l2,r2][l1,r1]×[l2,r2], where lili and riri (li<rili<ri) are nonnegative integers. Please find the minimum number of nodes that you must choose in order to make the union of regions accounted for by the chosen nodes exactly the same as the given region. Here, two sets of points are considered different if there exists a point included in one but not in the other.

∗∗Regions are sets of points with real coordinates, where the point (x,y)(x,y) is included in the region [p,q]×[r,s][p,q]×[r,s] if and only if p≤x≤qp≤x≤q and r≤y≤sr≤y≤s. Here, ×× formally refers to Cartesian product of sets.

Input

Each test contains multiple test cases. The first line contains the number of test cases tt (1≤t≤1041≤t≤104). The description of the test cases follows.

The only line of each test case contains four integers l1l1, r1r1, l2l2, r2r2 — the boundaries of the region in each axis (0≤li<ri≤1060≤li<ri≤106).

Output

For each test case, output the minimum number of nodes necessary to satisfy the condition on a separate line.

Note

On the first test case, the given region is [0,1]×[1,2][0,1]×[1,2]. There is one node accounting for [0,1]×[1,2][0,1]×[1,2]. Choosing this node, the answer is 11.

On the second test case, the given region is [0,2]×[0,2][0,2]×[0,2]. There is one node accounting for [0,2]×[0,2][0,2]×[0,2]. Choosing this node, the answer is 11.

On the third test case, the given region is [1,3]×[1,3][1,3]×[1,3]. There is no node that accounts for [1,3]×[1,3][1,3]×[1,3]. Instead, you can make the union of regions exactly the same as [1,3]×[1,3][1,3]×[1,3] by choosing the following 44 nodes:

  • A leaf node accounting for [1,2]×[1,2][1,2]×[1,2];
  • A leaf node accounting for [1,2]×[2,3][1,2]×[2,3];
  • A leaf node accounting for [2,3]×[1,2][2,3]×[1,2];
  • A leaf node accounting for [2,3]×[2,3][2,3]×[2,3].

It can be shown that it is impossible to make the union of regions exactly the same as [1,3]×[1,3][1,3]×[1,3] with less than 44 nodes. Therefore, the answer is 44.

On the fourth test case, the given region is [0,2]×[1,5][0,2]×[1,5]. You can make the union of regions exactly the same as [0,2]×[1,5][0,2]×[1,5] by choosing the following 55 nodes:

  • A leaf node accounting for [0,1]×[1,2][0,1]×[1,2];
  • A leaf node accounting for [1,2]×[1,2][1,2]×[1,2];
  • A non-leaf node accounting for [0,2]×[2,4][0,2]×[2,4];
  • A leaf node accounting for [0,1]×[4,5][0,1]×[4,5];
  • A leaf node accounting for [1,2]×[4,5][1,2]×[4,5].

It can be shown that it is impossible to make the union of regions exactly the same as [0,2]×[1,5][0,2]×[1,5] with less than 55 nodes. Therefore, the answer is 55.
