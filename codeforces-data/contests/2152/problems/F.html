F. Triple Attack

3 seconds

1024 megabytes
input
standard input

Zeus is analyzing a replay of the fight to understand his opponent's attack patterns. The opponent has a special ability: if they land three attacks on a target within a time frame of zz, their third attack becomes a powerful, boosted attack.

To outplay his opponent, Zeus should not let his opponent trigger their boosted attack. Let Y={y1,y2,…,ym}Y={y1,y2,…,ym} be the multiset of mm timestamps, where each yiyi represents the time when the opponent's attack landed. We call YY to be safe if for every three timestamps {yi,yj,yk}{yi,yj,yk} such that 1≤i<j<k≤m1≤i<j<k≤m, it holds that max(yi,yj,yk)−min(yi,yj,yk)>zmax(yi,yj,yk)−min(yi,yj,yk)>z, where zz is the duration of the time frame that is given to you as an input.

Zeus has a log of nn timestamps, x1,x2,…,xnx1,x2,…,xn, representing when the opponent's attacks landed. The timestamps are sorted in nondecreasing order of occurrence. In other words, xi≤xi+1xi≤xi+1 for all 1≤i<n1≤i<n.

Zeus has qq intervals of interest, denoted as two integers 1≤l≤r≤n1≤l≤r≤n. For each interval, Zeus wants to find the maximum number of attacks among [xl,xl+1,…,xr][xl,xl+1,…,xr] that he could have let through: In other words, Zeus wants to find a maximum size subset of the multiset {xl,xl+1,…,xr}{xl,xl+1,…,xr} such that the subset is safe.

Input

Each test contains multiple test cases. The first line contains the number of test cases tt (1≤t≤200001≤t≤20000). The description of the test cases follows.

The first line contains two integers nn and zz (1≤n≤2500001≤n≤250000, 1≤z≤1091≤z≤109).

The second line contains nn integers x1,x2,…,xnx1,x2,…,xn (1≤xi≤1091≤xi≤109) — the timestamps of the opponent's attacks. It is guaranteed that the array xx is sorted, i.e., xi≤xi+1xi≤xi+1 for all 1≤i<n1≤i<n.

The third line contains a single integer qq (1≤q≤2500001≤q≤250000).

The next qq lines each contain two integers ll and rr (1≤l≤r≤n1≤l≤r≤n) — the endpoints of the interval.

It is guaranteed that the sum of nn over all test cases does not exceed 250000250000.

It is guaranteed that the sum of qq over all test cases does not exceed 250000250000.

Output

For each of the qq queries, print a single integer — the maximum size of a safe subset of attack timestamps in the given interval.
