H. Wonderful XOR Problem
time limit per test
2 seconds
memory limit per test
256 megabytes
input
standard input
output
standard output

You are the proud... never mind, just solve this problem.

There are nn intervals [l1,r1],[l2,r2],…[ln,rn][l1,r1],[l2,r2],…[ln,rn]. For each xx from 00 to 2m−12m−1, find the number, modulo 998244353998244353, of sequences a1,a2,…ana1,a2,…an such that:

  • li≤ai≤rili≤ai≤ri for all ii from 11 to nn;
  • a1⊕a2⊕…⊕an=xa1⊕a2⊕…⊕an=x, where ⊕⊕ denotes the bitwise XOR operator.
Input

Each test contains multiple test cases. The first line contains the number of test cases tt (1≤t≤1041≤t≤104). The description of the test cases follows.

The first line contains two integers nn and mm (1≤n≤2⋅1051≤n≤2⋅105, 1≤m≤181≤m≤18).

The ii-th of the next nn lines contains two integers lili and riri (0≤li≤ri<2m0≤li≤ri<2m).

It is guaranteed that the sum of nn over all test cases does not exceed 2⋅1052⋅105, and the sum of 2m2m over all test cases does not exceed 218218.

Output

For each xx from 00 to 2m−12m−1, let:

  • fxfx be the number of valid sequences, modulo 998244353998244353;
  • gx=fx⋅2xmod998244353gx=fx⋅2xmod998244353.

Here, fxfx and gxgx are both integers in the interval [0,998244352][0,998244352].

Let h=g0⊕g1⊕…⊕g2m−1h=g0⊕g1⊕…⊕g2m−1.

Output a single integer — the value of hh itself. Do not perform a modulo operation.
