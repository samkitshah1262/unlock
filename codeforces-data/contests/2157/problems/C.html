You are given three positive integers nn, kk, and qq. You are also given qq tuples (c,l,r)(c,l,r), with 1≤c≤21≤c≤2 and 1≤l≤r≤n1≤l≤r≤n.

An array a1,a2,…,ana1,a2,…,an is meximum if 0≤ai≤1090≤ai≤109 for each ii in [1,n][1,n], and for each given tuple (c,l,r)(c,l,r),

  • if c=1c=1, then min(al,al+1,…,ar)=kmin(al,al+1,…,ar)=k;
  • if c=2c=2, then MEXMEX∗∗(al,al+1,…,ar)=k(al,al+1,…,ar)=k.

Note that the parameter kk is the same for all the conditions.

Find a meximum array a1,a2,…,ana1,a2,…,an of length nn. The input is generated in such a way that a valid array always exists. If there are multiple possible arrays, you can print any one of them.

∗∗The minimum excluded (MEX) of a collection of integers a1,a2,…,aka1,a2,…,ak is defined as the smallest non-negative integer xx which does not occur in the collection aa.

Input

Each test contains multiple test cases. The first line contains the number of test cases tt (1≤t≤5001≤t≤500). The description of the test cases follows.

The first line of each test case contains three integers nn, kk, qq (1≤k≤n≤1001≤k≤n≤100, 1≤q≤1001≤q≤100) — the length of the array a1,a2,…,ana1,a2,…,an, the result of the minmin and MEXMEX calculations described by the tuples, and the number of tuples.

Then, qq lines follow. The ii-th line contains a tuple (c,l,r)(c,l,r), which gives a constraint to al,al+1,…,aral,al+1,…,ar according to the statement.

It is guaranteed that there exists a valid array corresponding to the given input.

Note that there are no constraints on the sum of nn, kk, or qq over all test cases.

Output

For each test case, print a single line containing a meximum array a1,a2,…,ana1,a2,…,an.
