2157A - Dungeon Equilibrium

Author: TheScrasse
Preparation: TheScrasse

Hint 1

Each value xx is independent of the others; only its own frequency matters.

Hint 2

In a balanced array, value xx must appear either 00 times or exactly xx times.

Solution
  • Each value xx is independent of the others; only its own frequency matters. In particular, for each value, find its frequency f[x]f[x].
  • In a balanced array, value xx must appear either 00 times or exactly xx times. Specifically, if it appears <x<x times at the beginning, you have to delete all its occurrences, otherwise you should keep xx occurrences. Specifically, for each xx, add f[x]f[x] to the answer if f[x]<xf[x]<x, else add f[x]−xf[x]−x.

Complexity: O(n)O(n) time. Slower solutions (e.g., O(n2)O(n2)) are allowed.

2157B - Expansion Plan 2

Author: TheScrasse
Preparation: TheScrasse

Hint 1

The order of operations does not matter. Only the number of 4s4s and 8s8s in the substring matters.

Hint 2

The black cells are the positions we can reach starting from (0,0)(0,0), if we can move orthogonally (using a 44) and diagonally (using an 88).

Hint 3

A move with an 88 is equivalent to two moves with a 44, in different directions.

Solution
  • The order of operations does not matter. So count the number of 4s4s and 8s8s, and assume there are aa 44s and bb 8s8s.
  • The grid is symmetrical. We can assume x,y≥0x,y≥0 (by making x:=|x|x:=|x|, y:=|y|y:=|y|).
  • The black cells are the positions we can reach starting from (0,0)(0,0), if we can move orthogonally (using a 44) and diagonally (using an 88).
  • In particular, since x,y≥0x,y≥0, a 44 is a step either up or right while an 88 consists of two steps, one up and one right.
  • So you make a+2ba+2b steps in total, and only a+ba+b of them can be in the same direction.
  • The necessary and sufficient conditions are
    • a+2b≥x+ya+2b≥x+y (you need to be able to perform x+yx+y steps, either up or right, to reach (x,y)(x,y));
    • a+b≥max(x,y)a+b≥max(x,y) (you need to be able to perform yy steps up and xx steps right).

2157C - Meximum Array 2

Author: TheScrasse
Preparation: TheScrasse

Hint 1

For each element, determine whether it belongs to a MEXMEX constraint, a minmin constraint, both, or none.

Hint 2

If an element belongs to both a MEXMEX constraint and a minmin constraint, it is useless for both constraints.

Hint 3

If an element only belongs to a minmin constraint, you can set it to kk for convenience.

Hint 4

What's the "fastest" way to satisfy a MEXMEX constraint? What about several constraints simultaneously?

Solution

For each element, determine whether it belongs to a MEXMEX constraint, a minmin constraint, both, or none.

  • If an element is covered by a minmin constraint, it must be ≥k≥k, and ideally it should be =k=k (so that the constraint is immediately satisfied).
  • If an element is covered by a MEXMEX constraint, it must be ≠k≠k, and ideally it should be <k<k (to make the constraint easier to satisfy).

Then

  • If an element is not covered by any constraints, it does not matter, and you can set it to any value.
  • If an element belongs to both a MEXMEX constraint and a minmin constraint, it is useless for both constraints, but you must set it to something both ≠k≠k and ≥k≥k. For example, you can set it to k+1k+1.
  • If an element only belongs to a minmin constraint, it is optimal to set it to kk.
  • The remaining elements are the ones which are only covered by a MEXMEX constraint. In order to be satisfied, a MEXMEX constraint must contain at least kk such elements (because it must contain all the elements between 00 and k−1k−1, which would violate a minmin constraint). In particular, assigning i mod ki mod k at the ii-th such element works (if a MEXMEX constraint contains kk such elements, their values are [0,1,…,k−1][0,1,…,k−1], possibly shifted).

Complexity: O(nq)O(nq) or O(n+q)O(n+q) per test case.

Bonus: in the above solution, we assign i mod ki mod k to the ii-th element only covered by MEXMEX constraints. However, assigning ai=i mod kai=i mod k to such elements (where ii is the position in the whole array, instead of the position only considering such elements) also works. Can you prove it?

2157D - Billion Players Game

Author: TheScrasse
Preparation: TheScrasse

Hint 1

The worst case is either x=lx=l or x=rx=r. So you can assume that xx is either ll or rr, but you don't know which one.

Hint 2

There exists an optimal strategy where a prefix of offers (after sorting) is accepted using the third option, and a suffix of elements is accepted using the second option.

Hint 3

There exists an optimal strategy where (in addition to the previous properties) at most one offer is rejected.

Solution

The worst case is either x=lx=l or x=rx=r.
Sketch of proof. After fixing the strategy, the number of coins as a function of xx is a straight line.
Then, in order to test a strategy, it is enough to evaluate it at x=lx=l and x=rx=r and pick the minimum.

There exists an optimal strategy where a prefix of elements (after sorting) is red, and a suffix of elements is blue.
Sketch of proof. Use exchange argument. For example, if y≤zy≤z, yy is white and zz is red, then we get x−zx−z coins. If we make yy red and zz white, we make x−yx−y coins instead.

There exists an optimal strategy where (in addition to the previous properties) at most one element is white.
Sketch of proof. If x≤yx≤y, and they are both white, we can make xx red and yy blue instead.

  • Try all possibilities for the white element. So there are O(n)O(n) candidate strategies.
  • We can test a strategy in O(1)O(1): find the number of coins as a function of xx using prefix sums, and evaluate it in ll and rr to check the actual worst case.

Complexity: O(nlogn)O(nlog⁡n)

2157E - Adjusting Drones

Author: TheScrasse
Preparation: TheScrasse

Hint 1

The order of the elements does not matter.

Hint 2

Solve the problem with k=1k=1.

Hint 3

Iterate over elements from right to left and find their final value if k=1k=1.

Hint 4

Find the value of each element over time. Can you calculate the value of some element after a fixed number mm of operations?

Hint 5

Binary search the answer.

Solution

The order of the elements does not matter. So let's sort the elements in increasing order. Also, we have the freedom to choose the tiebreaker rule (i.e., which elements are increased by 11).

  • Assume the process stops when all the elements are distinct (i.e., k=1k=1). Let's find the configuration b1,b2,…,bnb1,b2,…,bn at that point. If some elements are equal, the rightmost does not increase.
  • From right to left, if the initial value is aiai, then bibi is the smallest x≥aix≥ai not present in bb yet.
  • If we look at a single element over time, it has values ai,ai+1,…,bi,bi,…,biai,ai+1,…,bi,bi,…,bi.

  • The maximum number of occurences of one element does not increase after one operation (if value xx appeared ≤k≤k times, it produces ≤k−1≤k−1 copies of x+1x+1, and at most one other copy of x+1x+1 survives).
  • So we can binary search the answer. Using the process described earlier, we can retrieve the configuration after mm moves in O(n)O(n).

Complexity: O(nlogn)O(nlog⁡n) time (there also exist "magical" O(n)O(n) solutions!)

2157F - Git Gud

Author: TheScrasse
Preparation: TheScrasse

Hint 1

How would you write the checker for this problem? (How to check efficiently whether a submission gets AC or WA)?

Hint 2

After you choose yy, calculating the optimal ll is easy. From now, let's denote the operations only using yy.

Hint 3

Consider some naive strategies: for example, [1,2,…,n−1][1,2,…,n−1], or [n−1,n−2,…,1][n−1,n−2,…,1]. Why are they bad? How to improve them?

Hint 4

You can fix your skill modulo 22, then modulo 44, then modulo 88, etc., but that's still a bit too inefficient.

Hint 5

Try with a different modulo.

Solution

As a warm-up, think about how we would write the checker for this problem.

We can store the set of all the possible skills (initially, all integers in [1,n][1,n]), and perform operations in O(1)O(1) (we have to remove yy and insert y+ly+l).

As a consequence, the optimal value of ll is the smallest one such that y+ly+l is already in the set. Since we know how to determine the optimal ll for a fixed yy, let's denote the operations only using yy.

  • Naive strategy 1: use operations 1,2,…,n−11,2,…,n−1. Unfortunately, they are too "increasing".
  • Naive strategy 2: use operations n−1,n−2,…,1n−1,n−2,…,1. Unfortunately, they are too long.

Better strategy:

  • Fix our skill modulo 22 using operations n−1,n−3,n−5,…n−1,n−3,n−5,…
  • Fix our skill modulo 44 using operations n−2,n−6,n−10,…n−2,n−6,n−10,…
  • ……

The cost is O(nlogn)O(nlog⁡n) (≈log2n≈log2⁡n layers, O(n)O(n) per layer). We use very few "increasing" operations (11 per layer): we can try to use more (and maybe use less layers).

Generalization:

  • Fix our skill modulo 33 using operations n−2,n−5,n−8,…,n−1,n−4,n−7,…n−2,n−5,n−8,…,n−1,n−4,n−7,…
  • Fix our skill modulo 99 using operations n−6,n−18,n−30,…,n−3,n−12,n−21,…n−6,n−18,n−30,…,n−3,n−12,n−21,…
  • ……

Now we have ≈log3n≈log3⁡n layers and 22 "increasing" operations per layer.

If we use modulo mm, we have ≈logmn≈logm⁡n layers, each with cost ≈n≈n, and m−1m−1 "increasing" operations per layer, each with cost 10001000.

We should choose m≈n−−√3m≈n3. So the total cost is around

(logmn)(n+1000m)=3(n+1000n−−√3)<4n(logm⁡n)(n+1000m)=3(n+1000n3)<4n

if $n = 250\,000$.

2157G - Isaac's Queries

Author: KLPP
Preparation: KLPP

Hint 1

Apply prefix XOR. So, instead of information about al⊕al+1⊕…⊕aral⊕al+1⊕…⊕ar, assume you can ask information about al⊕aral⊕ar.

Hint 2

Find whether the highest bit of every aiai (viewed as 3030-bit integer) is 00 or 11. You can assume that a1a1 has highest bit 00.

Hint 3

You get two blocks. Recurse.

Hint 4

Solve every block in the most efficient way possible.

Solution

Apply prefix XOR. So, instead of information about al⊕al+1⊕…⊕aral⊕al+1⊕…⊕ar, we assume we can ask information about al⊕aral⊕ar.

Let's build a trie containing the aiai. Specifically, each node corresponds to a prefix of the binary representation, and contains positions ii such that aiai has that prefix. It is not possible to recover the aiai uniquely, but we can set bits to 00 without loss of generality if they do not impact any queries. Then, in order to calculate the answer for some (l,r)(l,r), we have to find the LCA of the corresponding nodes.

Suppose we are in some node of the trie containing kk values. We want to recurse into two children, depending on the next bit. This is possible by asking the query.

Specifically, suppose that a node contains positions p1,p2,…,pkp1,p2,…,pk. In order to recurse, we need to ask queries between some pairs. We choose this pairs greedily, by calculating a minimum spanning tree (the edges have cost 1pj−pi+11pj−pi+1). It can be also shown that, in the minimum spanning tree, the picked edges contain either p1p1 or pkpk, which makes the implementation easier.

Empirically, this solution requires around 88 robocoins per test case with n=100n=100.

Bonus: prove the asymptotic behavior of the solution (idea: for each (i,j)(i,j), estimate the probability that the query is performed).

2157H - Keygen 3

Author: TheScrasse
Preparation: TheScrasse, Dominater069

Hint 1

Start with some "easy" pairs (n,m)(n,m). Which ones can you solve?

Hint 2

If you have a solution to (n,m)(n,m), can you transform it into a solution to (n+1,m+1)(n+1,m+1)?

Hint 3

If you have a solution to (n,m)(n,m), can you transform it into a solution to (n+1,m)(n+1,m)?

Hint 4

Find small (n,m)(n,m) with at least 20002000 solutions, and propagate these solutions to some bigger pairs.

Hint 5

Now some pairs are missing. They either have small nn, or small n−mn−m.

Hint 6

If n−mn−m is small, what can you say about the amount of pi=ipi=i?

Hint 7

If n−mn−m is small, then pnpn must be large.

Solution

For convenience, let's find "anti-bitonic" permutations instead (decreasing, then increasing). There is a bijection between bitonic and anti-bitonic permutations.

vector<int> f(vector<int> p) {
    int n = p.size();
    reverse(p.begin(), p.end());
    for (auto &u : p) u = n - u + 1;
    return p;
}

We can transform a solution to (n,m)(n,m) into a solution to (n+1,m+1)(n+1,m+1): just append n+1n+1 at the end.

We can also transform a solution to (n,m)(n,m) into a solution to (n+1,m)(n+1,m): append n+1n+1 at the end, and swap p1p1 with p1+1p1+1.

Let's divide into three cases:

  1. n≤18n≤18;
  2. n≥19n≥19, n−m≥10n−m≥10;
  3. n≥19n≥19, n−m≤9n−m≤9.

They can be solved as follows.

  1. We can find all the anti-bitonic permutations and test them in O(n⋅2n)O(n⋅2n).
  2. First, solve (18,i)(18,i) for 1≤i≤81≤i≤8 (for each ii, we have ≥2000≥2000 solutions). Transform these solutions into solutions to (n,m)(n,m).
  3. p1,p2,…,pnp1,p2,…,pn must have at least n−9n−9 cycles, so it must have at least n−18n−18 fixed points (i.e., pi=ipi=i). The permutation structure is [p1,…,p1+1,p1+2,…,n][p1,…,p1+1,p1+2,…,n]: the highlighted square has at most 11 fixed point, so there are at most n−p1+1n−p1+1 fixed points. Therefore, p1≤19p1≤19, and we can iterate over all such permutations.

2157I - Hyper Smawk Bros

Author: TheScrasse
Full solution: dario2994
Preparation: TheScrasse, Dominater069

Hint 1

Solve the problem with even mm.

Hint 2

Solve the problem with t=1t=1.

Hint 3

For a fixed nn, how do "losing" mm look like?

Hint 4

Let (n,l)(n,l) (0≤m0≤m) be losing if you lose if it's your turn, the current integer is nn, and you cannot subtract ll.
If (n,l)(n,l) is losing, then (n+l,l′)(n+l,l′) with l′≠ll′≠l is winning. This means that the number of losing ll for a fixed nn is either 00, 11, or m+1m+1. Let's call nn "good", "neutral", and "evil", respectively.

Hint 5

How do the losing states (n,l)(n,l) look like?

Hint 6

Our goal is to solve t=1t=1 in O(n/m)O(n/m).

Hint 7

How to determine efficiently whether some nn is good or neutral?

Hint 8

In order to determine whether some nn is good or neutral, you can write a recursive function which terminates in O(1)O(1) on average.

Solution

Losing positions.

Let (n,l)(n,l) (0≤m0≤m) be losing if you lose if it's your turn, the current integer is nn, and you cannot subtract ll (if ll is 00, you can subtract everything).

Evil nn.

Let nn be evil if (n,0)(n,0) is losing (i.e., you lose even without constraints on the number you subtract).

Bound on the evil nn (I).

There are O(N/m)O(N/m) evil nn. Specifically, let v1,v2,…,vkv1,v2,…,vk be the evil n≤Nn≤N. Then, vi−vi−1≥m+1vi−vi−1≥m+1 (otherwise you would be able to move from an evil nn to another, contradiction).

Structure of losing positions and solution in O(t(N+m))O(t(N+m)).

If (n,l)(n,l) is losing, then (n+l,l′)(n+l,l′) with l′≠ll′≠l is winning. This means that the number of losing ll for a fixed nn is either 00, 11, or m+1m+1. Let's call nn "good", "neutral", and "evil", respectively.
Note that the losing states are O(N+m)O(N+m) in total.

Even mm.

For even mm, only the multiples of m+1m+1 are evil. In order to win, you can ensure that the last two moves have sum m+1m+1. Since m+1m+1 is odd, the last move will never be the same as the second last.

From now, let's only consider odd mm.

More structure.

Let f(n)f(n) be the distance from the largest evil n′≤nn′≤n.

There are three types of losing states.

  • (n,l)(n,l) with f(n)=0f(n)=0.
  • (n,f(n))(n,f(n)).
  • (n,f(n)/2)(n,f(n)/2) with f(n)=m+1f(n)=m+1.

This is a consequence of the slow solution described above. If ss is evil:

  • the losing states (s,l)(s,l) propagate to nn in [s+1,s+m][s+1,s+m];
  • only the losing state (s+m+12,m+12)(s+m+12,m+12) can propagate to (s+m+1,m+12)(s+m+1,m+12), and in that case s+m+2s+m+2 is evil.

So s+m+1s+m+1 cannot be good (it is either neutral or evil).

Bound on the evil nn (II).

Let v1,v2,…,vkv1,v2,…,vk be the evil n≤Nn≤N. Then, m+1≤vi−vi−1≤m+2m+1≤vi−vi−1≤m+2.

Our goal.

Let v1,v2,…,vkv1,v2,…,vk be the evil n≤Nn≤N. We want to find them in O(k)=O(N/m)O(k)=O(N/m). In this way, we can solve the problem for all mm in O(∑1≤m≤MN/m)=O(NlogM)O(∑1≤m≤MN/m)=O(Nlog⁡M).

Finding vi+1vi+1.

We have v1,v2,…,viv1,v2,…,vi, and we want to find vi+1vi+1 in O(1)O(1). It is enough to check whether vi+m+1vi+m+1 is evil. It's easier to check whether it is not evil, i.e., whether there exists a winning move.
Let's start from some nn which satisfies vj≤n≤vj+1vj≤n≤vj+1. A winning move must ensure that the opponent cannot move to an evil state (necessary condition). So there are three candidate winning moves.

  • Subtracting n−vj+1n−vj+1 (so that the opponent is too far from vj−1vj−1; this can only work if vj−vj−1=m+2vj−vj−1=m+2).
  • Subtracting n−vj2n−vj2 (so that the opponent would have to subtract n−vj2n−vj2 again to reach vjvj).
  • Subtracting n−vj−12n−vj−12 (so that the opponent would have to subtract n−vj−12n−vj−12 again to reach vj−1vj−1).

Finding vi+1vi+1 efficiently.

The previous observation provides a recursive function which can call itself up to 33 times.
The recursive function answers the question "can we win starting from (n,n−vj)(n,n−vj)?" (i.e., it is forbidden to make the move which would win immediately). This is equivalent to "is nn good or neutral?"
We want to make the average number of recursive calls less than 11, so that the function terminates in O(1)O(1).

  • We do not need the first call, because vj−1+m+1vj−1+m+1 is neutral (proved before).
  • The second call happens with probability ≲1/2≲1/2 (n−vjn−vj must be even, and the first case must not apply).
  • The third call happens with probability ≲3/8≲3/8 (n−vj−1n−vj−1 must be even, the first case must not apply, and the second call must not succeed).

So the function calls itself ≲7/8≲7/8 times on average. This is fast enough for all mm (it can be tested locally).
