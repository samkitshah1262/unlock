D. MAD Interactive Problem
time limit per test
2 seconds
memory limit per test
256 megabytes
input
standard input
output
standard output

This is an interactive problem.

There is a secret sequence a1,a2,…,a2n−1,a2na1,a2,…,a2n−1,a2n, which contains each integer from 11 to nn exactly twice.

Your task is to guess the sequence by using queries of the following type:

  • "? kj1j2…jkkj1j2…jk" — select the integer kk (1≤k≤2n1≤k≤2n) and kk distinct indices j1,j2,…,jkj1,j2,…,jk (1≤j1,j2,…,jk≤2n1≤j1,j2,…,jk≤2n). In response to the query, the jury will return MAD([aj1,aj2,…,ajk])MAD([aj1,aj2,…,ajk]).

We define the MADMAD (Maximum Appearing Duplicate) of an integer sequence as the largest integer that appears at least twice. Specifically, if there is no number that appears at least twice, the MADMAD value is 00. Some examples are as follows:

  • MAD([1,2,1])=1MAD⁡([1,2,1])=1;
  • MAD([2,2,3,3])=3MAD⁡([2,2,3,3])=3;
  • MAD([1,2,3,4])=0MAD⁡([1,2,3,4])=0.

Please identify the secret sequence using at most 3n3n queries.

Input

Each test contains multiple test cases. The first line contains the number of test cases tt (1≤t≤30001≤t≤3000). The description of the test cases follows.

The first line of each test case contains one integer nn (2≤n≤3002≤n≤300).

It is guaranteed that the sum of n2n2 over all test cases does not exceed 105105.

After you read this line of input, the interaction begins with your first query.

Interaction

To make a query, output a line (without quotes) in the following format:

  • "? kj1j2…jkkj1j2…jk" (1≤k≤2n1≤k≤2n, 1≤j1,j2,…,jk≤2n1≤j1,j2,…,jk≤2n)

Here, the indices j1,j2,…,jkj1,j2,…,jk which you selected must be pairwise distinct.

Then, after each query, read a single integer — the answer to your query.

You can make at most 3n3n queries of this type.

If your program has found the sequence aa, print the line (without quotes) in the following format:

  • "! a1a2…a2n−1a2na1a2…a2n−1a2n" (1≤ai≤n1≤ai≤n)

Note that this does not count towards the query limit.

After this, proceed to the next test case, or exit if this is the last test case.

The interactor in this task is not adaptive. In other words, the sequence aa does not change during the interaction.

If you make more than 3n3n queries during an interaction, your program must terminate immediately, and you will receive the Wrong Answer verdict. Otherwise, you can get an arbitrary verdict because your solution will continue to read from a closed stream.

After printing each line, do not forget to output the end of line and flush the output buffer. Otherwise, you will receive the Idleness Limit Exceeded verdict. To flush, use:

  • fflush(stdout) or cout.flush() in C++;
  • System.out.flush() in Java;
  • stdout.flush() in Python;
  • see the documentation for other languages.

Hacks

To make a hack, please use the following format:

The first line should contain the number of test cases tt (1≤t≤30001≤t≤3000).

The first line of each test case should contain an integer nn (2≤n≤3002≤n≤300).

The following line should contain 2n2n integers a1,a2,…,a2na1,a2,…,a2n (1≤ai≤n1≤ai≤n). Each number from 11 through nn must appear exactly twice.

The sum of n2n2 over all test cases should not exceed 105105.
