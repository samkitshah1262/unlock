2168B - Locate

Editorial

Let's try to solve the problem player B has to solve.

Let the range of a subarray be max(pl,pl+1,...,pr)−min(pl,pl+1,...,pr)max(pl,pl+1,...,pr)−min(pl,pl+1,...,pr). A subarray has range n−1n−1 iff it contains both 11 and nn. We can binary search on the shortest prefix with range n−1n−1, this will find the latter position of 11 or nn. Also binary search on the shortest suffix with range n−1n−1, this will find the former position of 11 or nn. Now we've found the position of 11 and nn, but we don't know which is which.

This is where player A comes in. If 11 appears before nn, he will send 00; otherwise, he will send 11. Now player B can figure out which is which.

2168C - Intercepting Butterflies

Editorial

Let's phrase the problem slightly differently.

  • Player A has a binary string of length 1515, and he will send a binary string of length 2020.
  • On the way, none or one of the bits get flipped.
  • Given the altered string, BB must find the original binary string of length 1515.

The first 1515 bits (s1..15s1..15) will be allocated for the original string, and the remaining 55 bits will be used to reconstruct where the flip happened.

The next 44 (s15..19s15..19) will house the bitwise XOR of the all the positions in the original string that are 1. The last bit (s20s20) will be the number of bits in previous 44 bits mod 22.

Now there are 22 cases:

Case 1: A flip happens in s16..20s16..20

We can detect this case by checking if the last bit matches what it's supposed to be. If it doesn't, a flip has happened somewhere in the last 55 bits. In this case, the original 1515 are completely untouched.

Case 2: A flip doesn't happen or happens in s1..15s1..15

Here, we calculate the bitwise XOR of all the 1 in s1..15s1..15. Let xx be this value and yy be the number stored in s16..19s16..19. The cool thing is: x⨁yx⨁y will be the position of the flipped bit, or 00 if no flip occurred. This is because if a flip occured at position kk, xx becomes x⨁kx⨁k.
