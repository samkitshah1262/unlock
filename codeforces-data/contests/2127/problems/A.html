You are given an array aa consisting of nn non-negative integers. However, some elements of aa are missing, and they are represented by −1−1.

We define that the array aa is good if and only if the following holds for every 1≤i≤n−21≤i≤n−2:

mex([ai,ai+1,ai+2])=max([ai,ai+1,ai+2])−min([ai,ai+1,ai+2]),mex⁡([ai,ai+1,ai+2])=max([ai,ai+1,ai+2])−min([ai,ai+1,ai+2]),

where mex(b)mex⁡(b) denotes the minimum excluded (MEX)∗∗ of the integers in bb.

You have to determine whether you can make aa good after replacing each −1−1 in aa with a non-negative integer.

∗∗The minimum excluded (MEX) of a collection of integers b1,b2,…,bkb1,b2,…,bk is defined as the smallest non-negative integer xx which does not occur in the collection bb. For example, mex([2,2,1])=0mex⁡([2,2,1])=0 because 00 does not belong to the array, and mex([0,3,1,2])=4mex⁡([0,3,1,2])=4 because 00, 11, 22, and 33 appear in the array, but 44 does not.

Input

Each test contains multiple test cases. The first line contains the number of test cases tt (1≤t≤5001≤t≤500). The description of the test cases follows.

The first line of each test case contains a single integer nn (3≤n≤1003≤n≤100) — the length of aa.

The second line contains nn integers a1,a2,…,ana1,a2,…,an (−1≤ai≤100−1≤ai≤100) — the elements of aa. ai=−1ai=−1 denotes that this element is missing.

Output

For each test case, output "YES" if it is possible to make aa good, and "NO" otherwise.

You can output the answer in any case (upper or lower). For example, the strings "yEs", "yes", "Yes", and "YES" will be recognized as positive responses.

Note

In the first test case, we can put a1=a2=a3=1a1=a2=a3=1. Then,

  • mex([a1,a2,a3])=mex([1,1,1])=0mex⁡([a1,a2,a3])=mex⁡([1,1,1])=0;
  • max([a1,a2,a3])=max([1,1,1])=1max([a1,a2,a3])=max([1,1,1])=1;
  • min([a1,a2,a3])=min([1,1,1])=1min([a1,a2,a3])=min([1,1,1])=1.

And 0=1−10=1−1. Thus, the array aa is good.

In the second test case, none of the elements in aa is missing. And we have

  • mex([a1,a2,a3])=max([a1,a2,a3])−min([a1,a2,a3])mex⁡([a1,a2,a3])=max([a1,a2,a3])−min([a1,a2,a3]),
  • mex([a2,a3,a4])=max([a2,a3,a4])−min([a2,a3,a4])mex⁡([a2,a3,a4])=max([a2,a3,a4])−min([a2,a3,a4]), but
  • mex([a3,a4,a5])≠max([a3,a4,a5])−min([a3,a4,a5])mex⁡([a3,a4,a5])≠max([a3,a4,a5])−min([a3,a4,a5]).

Thus, the array aa cannot be good.

And 0≠5−10≠5−1. So the array aa cannot be good, no matter how you replace the missing elements.
