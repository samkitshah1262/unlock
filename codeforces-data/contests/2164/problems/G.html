G. Pointless Machine
time limit per test
4 seconds
memory limit per test
256 megabytes
input
standard input
output
standard output

This is an interactive problem.

Madeline is playing with a pointless machine. The pointless machine has a hidden tree of size nn, and Madeline has to find the edges of the tree by asking the machine.

In a query, Madeline will give the machine a permutation pp of [1,2,…,n][1,2,…,n], and the machine will return a sequence q1,q2,…,qnq1,q2,…,qn, where qiqi is the number of edges of the induced subgraph formed by the vertices {p1,p2,…,pi}{p1,p2,…,pi}.

Here, an induced subgraph of a graph G(V,E)G(V,E) formed by a subset of vertices V′⊆VV′⊆V, is a graph consisting of vertices from this subset and all edges between vertices from subset that are present in original graph GG.

However, the machine works slowly, so Madeline can only get the results at once after all queries are completed. The memory of this machine is not very large either, so Madeline can only ask 3131 times. She doesn't know how to solve it, so she invited you to help her.

Note that the interactor is non-adaptive. That is, tree is fixed in advance and doesn't change with your queries.

Input

Each test contains multiple test cases. The first line contains the number of test cases tt (1≤t≤1041≤t≤104). The description of the test cases follows.

The only line of each test case contains a single integer nn (3≤n≤5⋅1043≤n≤5⋅104) — the size of the tree.

It is guaranteed that the sum of nn over all test cases does not exceed 5⋅1045⋅104.

Interaction

The interaction begins by reading the integer nn.

Then, output a single integer kk (1≤k≤311≤k≤31) — the number of queries.

To ask a query, output a line in the following format:

  • pi,1pi,2…pi,npi,1pi,2…pi,n (1≤pi,j≤n1≤pi,j≤n, the pi,jpi,j are distinct for each ii) — a query described above.

Once you've asked all kk queries, read kk lines of nn integers qi,jqi,j — the responses to the queries as described above.

When you know the hidden tree, output n−1n−1 lines in the following format:

  • u,vu,v (1≤u,v≤n1≤u,v≤n) — an edge (u,v)(u,v) in the hidden tree.

Then, move on to the next test case, or terminate the program if there are no more test cases.

After printing all kk queries do not forget to output the end of line and flush∗∗ the output. Otherwise, you will get Idleness limit exceeded verdict.

If, at any interaction step, you read −1−1 instead of valid data, your solution must exit immediately. This means that your solution will receive Wrong answer because of an invalid query or any other mistake. Failing to exit can result in an arbitrary verdict because your solution will continue to read from a closed stream.

Hacks

To hack, use the following format.

The first line should contain a single integer tt (1≤t≤1041≤t≤104).

The second line of each test case should contain a single integer nn (3≤n≤5⋅1043≤n≤5⋅104).

ii-th out of next n−1n−1 lines should contain two integers ui,viui,vi (1≤ui,vi≤n1≤ui,vi≤n).

Given edges should form a tree, and the sum of nn over all test cases shouldn't exceed 5⋅1045⋅104.

∗∗To flush, use:

  • fflush(stdout) or cout.flush() in C++;
  • sys.stdout.flush() in Python;
  • see the documentation for other languages.

For the first test case:

  • The first query has answer 0 1 2, since only edge (1,2)(1,2) is in the induced subgraph formed by {1,2}{1,2}.
  • The second query has answer 0 0 2, since no edges are in the induced subgraph formed by {1,3}{1,3}.

For the third test case:

  • Edges (2,3)(2,3), (3,4)(3,4), and (4,1)(4,1) are in the induced subgraph formed by {1,2,3,4}{1,2,3,4}, so q1,4=3q1,4=3.
