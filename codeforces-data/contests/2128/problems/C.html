C. Leftmost Below
time limit per test
2 seconds

256 megabytes
input
standard input

standard output

Consider an array a1,…,ana1,…,an. Initially, ai=0ai=0 for every ii.

You can do operations of the following form.

  • You choose an integer xx greater than min(a)min(a).
  • Then, ii is defined as the minimum index such that ai<xai<x. In other words, ii is the unique integer between 11 and nn inclusive such that ai<xai<x and aj≥xaj≥x for every 1≤j≤i−11≤j≤i−1.
  • Finally, aiai is incremented by xx.

For example, if a=[6,8,2,1]a=[6,8,2,1] and you choose x=6x=6, then ii will be equal to 33 (since a1≥6a1≥6, a2≥6a2≥6, and a3<6a3<6) and aa will become [6,8,8,1][6,8,8,1].

You can do as many operations as you want. Can you reach a target array b1,…,bnb1,…,bn?

Input

Each test contains multiple test cases. The first line contains the number of test cases tt (1≤t≤100001≤t≤10000). The description of the test cases follows.

The first line of each test case contains a single integer nn (2≤n≤2000002≤n≤200000).

The second line of each test case contains nn integers b1,b2,…,bnb1,b2,…,bn (1≤bi≤1091≤bi≤109).

The sum of nn over all test cases does not exceed 200000200000.

Output

For each test case, print YES if you can reach the target array and NO otherwise.

You can output the answer in any case (upper or lower). For example, the strings "yEs", "yes", "Yes", and "YES" will be recognized as positive responses.
