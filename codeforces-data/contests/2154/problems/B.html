B. Make it Zigzag
time limit per test
1.5 seconds
memory limit per test
256 megabytes
input
standard input
output
standard output

An arbitrary array of integers bb of length mm is considered awesome if for all ii (1≤i<m1≤i<m):

  • if ii is odd then bi<bi+1bi<bi+1 holds.
  • if ii is even then bi>bi+1bi>bi+1 holds.

In other words, the following inequality is true: b1<b2>b3<b4…b1<b2>b3<b4…

You are given an array of positive integers aa of length nn. You may do either of the following operations any number of times, in any order:

  • operation 11: select an integer ii (1≤i≤n1≤i≤n) and do: ai:=max(a1,…,ai)ai:=max(a1,…,ai), that is, replace aiai with the prefix max up to ii.
  • operation 22: select an integer ii (1≤i≤n1≤i≤n) and then decrease aiai by 11.

Determine the minimum number of times you need to do operation 22 to make aa awesome. Note that you do not need to minimise the number of times you perform operation 11.

Input

Each test contains multiple test cases. The first line contains the number of test cases tt (1≤t≤1041≤t≤104). The description of the test cases follows.

The first line of each test case contains an integer nn (2≤n≤2⋅1052≤n≤2⋅105) — the length of the array aa.

The second line of each test case contains nn integers a1,a2,…,ana1,a2,…,an (1≤ai≤1091≤ai≤109).

The sum of nn across all test cases does not exceed 2⋅1052⋅105.

Output

For each testcase, output the minimum cost to make aa awesome.
